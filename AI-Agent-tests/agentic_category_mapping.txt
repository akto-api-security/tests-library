OWASP Agentic Top 10 category remap (from PR #289 - change agentic test new category).

This file documents how older logical families of tests (derived from the legacy
`TopCategory`/subcategory naming like `BUSINESS_ALIGNMENT_*`, `HALLUCINATION_AND_TRUSTWORTHINESS_*`,
`SAFETY_*`, `SECURITY_*`, etc.) were mapped into the new OWASP Agentic Top 10 buckets
(`info.category.name`) as implemented in PR #289
([change agentic test new catgeory](https://github.com/akto-api-security/tests-library/pull/289/changes)).

The mapping below is inferred directly from the current `AI-Agent-tests/generated_tests/*.yml`
files by grouping on the first two segments of each `info.subCategory` and reading
the OWASP Agentic category (`info.category.name`). In other words, for each old
logical family prefix we list the OWASP Agentic Top 10 category it was migrated to.

Key OWASP Agentic category names:
- AGENT_GOAL_HIJACK
- TOOL_MISUSE_AND_EXPLOITATION
- IDENTITY_AND_PRIVILEGE_ABUSE
- AGENTIC_SUPPLY_CHAIN
- UNEXPECTED_CODE_EXECUTION
- MEMORY_AND_CONTEXT_POISONING
- INSECURE_INTER_AGENT_COMMUNICATION
- CASCADING_FAILURES
- HUMAN_AGENT_TRUST_EXPLOITATION
- ROGUE_AGENTS

Specific‑scope buckets (use only when the risk clearly matches):
- TOOL_MISUSE_AND_EXPLOITATION
- IDENTITY_AND_PRIVILEGE_ABUSE
- UNEXPECTED_CODE_EXECUTION
- INSECURE_INTER_AGENT_COMMUNICATION
- HUMAN_AGENT_TRUST_EXPLOITATION
- CASCADING_FAILURES

Notes
-----
- `BUSINESS_ALIGNMENT_*` and `HALLUCINATION_AND_TRUSTWORTHINESS_*` were not mapped 1:1
  to a single OWASP Agentic category; instead, each sub-family was remapped based on
  its vulnerability description (`info.description` / `wordLists.base_prompt`).
- The same logical family prefix may span multiple OWASP Agentic categories when the
  underlying tests exercise different risks (for example, some `SECURITY_CONTEXT_*`
  tests are about leakage/privilege abuse, others about memory/RAG poisoning).
